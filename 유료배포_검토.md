## 🔴 유료화 전 반드시 해결

### 1. 수익·배포 모델이 아예 없음 (가장 큰 공백)
- 현재는 정적 페이지 → **결제·계정·콘텐츠 잠금 장치 전무.** 부모 잠금은 클라이언트 곱셈 문제라 우회 가능(설정 접근 차단용으론 OK, 콘텐츠 과금엔 부적합).
- 모델을 먼저 정해야 나머지 작업이 정해짐:
  - **(A) 앱스토어 배포**: Android=TWA(Play), iOS=Capacitor/WKWebView 래핑 → 스토어 IAP/구독. 키즈 카테고리 심사 규정 엄격.
  - **(B) 웹 구독**: 계정(로그인) + 결제(토스페이먼츠/아임포트/Stripe) + **서버에서 콘텐츠 게이팅**. 정적앱만으론 잠금이 불가능(클라이언트는 다 뜯김).
  - **(C) B2B/기관 라이선스**: 유치원·학습지 제휴.
- 어느 쪽이든 **백엔드(계정·결제·진도 동기화)** 최소 한 겹이 필요.

### 2. 법적·컴플라이언스 문서 전무 (아동 대상이라 특히 민감)
- `LICENSE`, `개인정보처리방침`, `이용약관`, 환불정책 **하나도 없음.**
- 아동 대상 서비스: 국내 **개인정보보호법**상 만14세 미만은 **법정대리인 동의** 필요. 미국 대상이면 **COPPA**, EU면 **GDPR-K**.
- 국내 유료 판매 시 **사업자등록 + 통신판매업 신고**, **전자상거래법** 청약철회/환불 고지 의무.
- 스토어 키즈 카테고리(Apple Kids / Google Families): 제3자 데이터 수집·비인증 광고 SDK 금지 등.

### 3. 제3자 데이터 유출 지점 — Google Fonts CDN
- `styles.css`가 **Google Fonts를 런타임 로드**(Gowun Dodum·Jua·Gaegu) → 사용자(아동) IP가 구글로 전송. "아무것도 수집 안 함"이라 말할 수 없게 됨 + 오프라인 실패.
- **폰트를 자체 호스팅(self-host)** 으로 전환하면 프라이버시·오프라인 둘 다 해결(쉬운 개선). ✅ 다행히 이모지 Twemoji는 no-op 스텁이라 미로드, 다른 트래커·CDN 없음.

### 4. 호스팅을 상용 가능한 곳으로 이전
- 현재 **GitHub Pages(개인)**: 대역폭 소프트 캡 ~100GB/월, 저장 ~1GB, SLA 없음, 헤더 설정 불가, 상업적 대량 사용 비권장.
- 앱 1회 ≈ 83MB 오디오 → 사용자 늘면 한도·비용 리스크. **Cloudflare Pages/Netlify/Vercel** 등으로 이전(커스텀 도메인·보안헤더·CSP·대역폭).

## 🟠 강력 권장 (품질·신뢰)

### 5. 자산 경량화 & 진짜 오프라인
- 오디오 **83MB / 4,266개**(음절 MP3 1,422 × 3음성). 서비스워커는 ~20개만 precache → **오프라인 시 대부분 음성 안 나옴**(network-first라 온라인 의존).
- 개선: ① 음성 1종으로 축소(83→28MB), ② MP3 비트레이트↓/opus 변환, ③ 현재 막 커리큘럼 오디오만 on-demand precache → 아이패드에서 진짜 오프라인 동작.

### 6. 오류·크래시 관측성(Observability)
- **분석·에러추적 전무** → 실제 기기(특히 iOS TTS 변덕)에서 뭐가 깨지는지 알 방법 없음.
- 프라이버시 안전한 방식으로 최소한: 크래시/에러(Sentry, PII 스크럽) + 핵심 퍼널(자체호스팅 Plausible 등, 동의 기반). 키즈 규정 준수 필수.

### 7. 런타임/E2E 테스트 안전망
- 현재 테스트는 **정적 문자열 검사(`*_check.py`) 137개뿐** — SW 버전 하드코딩 등 취약, 실제 동작 회귀는 못 잡음.
- **Playwright 스모크**(인트로 재생·글자 마스터·진도 저장·딥링크) 몇 개만 있어도 배포 사고 방지.

### 8. 진도 백업/계정 (데이터 소실 리스크)
- 진도는 **localStorage 전용** → 캐시 삭제·기기 변경 시 소실. 유료 사용자는 클라우드 동기화/복구를 기대. (2번 계정 도입과 연결)

### 9. 미완성 막(콘텐츠 구멍) 마감
- ~~`PROGRESS/ARCHITECTURE` 기준 **3막 글자 공방(자모 결합)·8막 문장 읽기 엔진이 "데이터만 정의"** 상태.~~ **[정정 2026-07-12]** 실측 결과 3막(`openCombine`/`combine` 화면)·8막(`openStory`/`story` 화면) 모두 **구현·플레이 가능**(테스트 `episode_progress_check.py` 통과). 남은 것은 "엔진"이 아니라 **콘텐츠 표본**: 3막 음절 6개·8막 문장 6개로 얇음 → 확장 필요.

## 📚 콘텐츠 · UX (학습 설계) 심화 검토

> 잘 되어 있는 것(유지): 비처벌·관대한 오답(감점 없이 쉐이크+소리 힌트+정답 강조) · 멀티감각(그림·소리·따라쓰기·드래그 결합) · 오늘의 미션 3종+출석 streak 일일 구조 · 서사/캐릭터 동기부여 · 음성학적 난이도순 글자 배열(ㅁㄴㅇ…) · 빠른 성취 맛보기(아이/오이/엄마…).

### 콘텐츠·교수법 (학습 효과에 직결)
- 🔴 **진짜 복습·간격반복(SRS)이 없음** — 익힌 글자는 이후 다시 테스트되지 않음. recap의 "지난 글자도 기억나요? [글자]"는 **문구 표시일 뿐 재연습 아님**. 만4세 파지(retention)엔 인터리빙 복습이 핵심. → 지난 글자를 짝맞추기·소리퀴즈에 섞어 재등장, 주기적 '복습의 날', 뗀 글자 하루 뒤 1회 재확인.
- 🟠 **고정 50스텝 경로 · 적응 난이도 0** — 헤매는 아이도 추가 연습 없이 통과, 잘하는 아이도 못 건너뜀. → 소리퀴즈 N회 오답이면 추가 연습/예시 삽입, 잘하면 '빠른 길' 제안.
- 🟠 **마스터리가 얕음(met+matched+quizzed 각 1회)** — '관대'는 좋지만 너무 쉽게 '뗌' 처리. → 최소한 다음 접속 시 그 글자 소리퀴즈 1회 지연 재확인 후 확정.
- 🟠 **자음을 '이름(기역)' 중심으로 제시** — 읽기엔 음가(그/ㄱ소리)가 더 중요. 글자상세 부제·`jamoSay`가 자음은 이름 반환. → 최신 읽기교육 흐름(소릿값 우선)에 맞춰 퀴즈·상세에서 **음가 강조**(이름은 보조).
- 🟡 **모음 10개 선(先)학습 부담** — 자음 전 모음 10개를 먼저 다 배움. 맛보기로 완화되지만 '진짜 읽기'까지 김. → act1 축약 또는 가/나 결합을 더 일찍 노출.
- 🟡 **콘텐츠 다양성 유한** — 단어/문장 뱅크가 작아(예: WORDS 12개) 수 주 반복 시 식상. → 글자별 예시단어·그림 로테이션, 단어/문장 확장.
- 🟡 **3·8막 콘텐츠 얇음** — **[정정 2026-07-12]** 막 자체는 구현됨(위 §9 정정 참조). 다만 3막 음절 6개·8막 문장 6개로 표본이 적어 유료 커리큘럼 체감상 얇음 → 확장 권장.

### UX (사용 경험)
- 🟠 **부모 대시보드 부재** — 지금은 곱셈 게이트→설정뿐. 유료 부모는 '무엇을 뗐나·주간 진행·추천 페이스'를 기대. 리텐션·환불방지에 큰 가치.
- 🟠 **다자녀 프로필 없음** — 형제 한 진도 공유. 가정 단위 필수.
- 🟠 **오늘 목표 후 부드러운 종료/스크린타임 경계 없음** — 무한 반복 가능. "오늘은 여기까지, 내일 또 만나요" 넛지 + 선택적 일일 한도 → 부모 구매 유인.
- 🟡 **선(先)-읽기 아동 UX 감사** — 모든 화면이 '소리+아이콘'만으로 동작하는지, 어디서나 🔊 다시듣기, 버튼 포커스 시 음성 안내. (대체로 양호 — 텍스트-only 게이트 없는지 최종 점검)
- 🟡 **잔재 놀이터 화면 공존** → 이중 내비 혼란. 통합/격리.
- 🟡 **'이어서 하기' 명확화** — 복귀 시 오늘의 다음 단계로 크게 유도(배너는 있음, 큰 CTA 권장).
- 🟡 **오답 스캐폴딩 단계화** — 지금은 즉시 정답 공개. 반복 오답 시 보기 수 축소·힌트 단계화로 성취감 유지.
- 🟡 **성취의 내재화** — 스티커/보상(외적)이 풍부. '방금 진짜 단어를 읽었어!'(내적) 순간을 더 이르게·구분해 강조(맛보기 확장).
- 🟡 **접근성** — reduced-motion 옵션(반복 애니 많음).

## 추천 순서 (로드맵)
1. **수익 모델 결정**(A/B/C) → 이게 백엔드/계정 유무를 좌우.
2. 폰트 self-host + 호스팅 이전(빠르고 효과 큼).

4. 계정·결제·진도 동기화(모델에 따라) + 콘텐츠 게이팅.
5. 오디오 경량화 + 오프라인 precache.
6. 에러추적 + Playwright 스모크.
7. 미완성 막 마감 + 잔재 화면 정리 + 접근성.
