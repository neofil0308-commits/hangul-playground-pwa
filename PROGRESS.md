## 현재 상태

- 정적 PWA 형태의 한글 학습 앱입니다.
- 앱 핵심은 `index.html`의 HTML/JavaScript, `styles.css`의 스타일, `app-data.js`의 정적 데이터, `app-state.js`의 미션 상태 로직, `app-listen.js`의 듣고 찾기 로직, `app-router.js`의 화면 이동/메뉴 초기화, `app-adventure.js`의 지도/스토리 렌더링, `app-learning.js`의 글자/단어 학습 로직, `app-writing.js`의 쓰기/획순 로직, `app-episode.js`의 에피소드 진행/별빛 앨범 렌더링으로 분리되었습니다.
- 핵심 경험이 `한글 떼기 커리큘럼(별빛 우체국 8막 여정)`으로 확장되었습니다. 스토리 진행 = 실제 한글 학습 순서이며, 만 4세·자모 백지 기준으로 한 화 = 글자 1개, 관대한 익힘 판정, 빠른 성취 맛보기로 설계되었습니다.
- 첫 실행 시 `인트로 그림책`(6장)이 나옵니다. 손그림 SVG 일러스트(종이 그레인 질감, 일관된 하니 캐릭터)에 장면별 의미 있는 CSS 애니메이션이 들어가고, 하니가 edge-tts 신경망 음성(`audio/narr/intro1~6.mp3`)으로 읽어줍니다. 글 못 읽는 4세를 위해 그림+소리 중심, 줄글 스토리는 `부모님 보기`로 접힙니다.
- service worker는 개발 중 캐시 꼬임을 막기 위해 `network-first`로 동작합니다(온라인이면 항상 최신, 오프라인이면 캐시 폴백).
- 오디오 MP3 리소스와 PWA manifest/service worker가 포함되어 있습니다.
- 현재 중심 경험은 `하니의 한글 모험`입니다.
- iPad 기준 홈 화면, 한글 마을 지도, 오늘의 모험, 한글 마을 이야기, 이야기 줄기, 스토리 바이블, 하니 반응까지 1차 고도화가 완료되었습니다.
- 전체 정적 회귀 테스트는 `python -m pytest -q tests/*_check.py`로 실행합니다. 일반 `python -m pytest -q`는 파일명 규칙 때문에 테스트를 자동 발견하지 못합니다.

### #1 — 낱자 게임에 음절/이모지가 섞여 나옴 (2026-07-12)

- **증상**: 듣고 찾기(3막)에서 정답 `바`(음절)와 오답 `ㅂ`·`ㅣ`(낱자)가 한 문제에 섞여 출제. 복습 게임에도 음절 `바`가 낱자 후보 사이에 등장. 8막은 `📖`가 보기로 나오고 소리로는 문장 전체를 읽음.
- **원인**: 게임 코드가 **`todayLetter.ch`를 항상 낱자로 가정**. 실제 `ch` 값은 막마다 다름 —
  - `letter` 막(1·2·4·5·6): 낱자 1개 ✅
  - `combine` 막(3): **음절**(`바`)
  - `sentence` 막(8): **`📖`**(소리는 문장 전체)
  - `word` 막(7): `curLetterObj`에 분기가 없어 **날짜 해시 랜덤 낱자**로 폴백(막과 무관)
  - 추가로 `completeCombine`이 `scheduleReview(ep.ch)`로 **음절을 복습 큐에 등록** → 복습 오염.
- **수정**: 음절/문장은 **초성으로 분해**해 낱자로 환원(`listenLetterTarget`), 복습 큐는 **`isJamoCh` 가드**로 낱자만 허용(등록·시드·소비 3지점), 7막은 **대표 단어의 첫 낱자**를 오늘의 글자로.
- **재발 방지 규칙**:
  1. `todayLetter.ch`/`ep.ch`를 낱자로 쓰기 전에 **막 유형(`ep.type`)을 반드시 확인**하라. 낱자 전용 UI/게임은 `combine`·`sentence`·`word`에서 방어 분기(분해 or 스킵)를 둔다. 참고 선례: 글자 찾기 `openFind`가 `combine`/`sentence`면 `go('home')`로 빠짐([app-learning.js](app-learning.js)).
  2. **낱자 판정은 `isJamoCh`(CHO/JUNG 멤버십)로 단일화**. `ch.length===1`은 음절도 통과하므로 금지.
  3. 복습·게임 **풀에 넣는 지점과 꺼내는 지점 양쪽에 필터**를 둬 기존 저장본 오염까지 방어한다.
  4. 새 막/유형 추가 시 **`EPISODE_PATH` 전체를 로드해 각 게임의 대상이 유효 낱자인지 시뮬레이션**하는 검증을 돌린다.
