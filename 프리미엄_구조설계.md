# 프리미엄(1회 구매) 구조 설계 — 초안 v0.1

> 목표: 많은 사람이 **저렴·안정적**으로. 서버 없이 도는 정적 PWA에 맞는 **1회 구매(비소모성) 잠금 해제**.
> 이 문서는 '구조'만. 실제 코드/결제 연동은 구조 합의 후 진행.
> ⚠️ 표시 = 결정 필요.

---

## 1. 제품 구조 — 무료 체험 → 1회 구매로 전체 해제
- **무료(체험)**: 1~2막 = 모음의 빛 + 자음 친구들. 첫 글자~첫 단어 맛보기(아이/오이/엄마)까지 경험 → 가치 충분히 체감.
- **프리미엄(유료)**: 3막부터(글자 공방·받침·쌍자음·숨은 모음·단어·문장) 전체 여정.
- ⚠️ **결정 1 — 무료 경계**: 1~2막(추천) / 1막만 / 3막까지.
- ⚠️ **결정 2 — 부가 기능 무료/유료**: 복습(SRS)·부모 대시보드·다자녀 프로필을
  (a) 무료 유지(체험 매력↑, 추천) 또는 (b) 프리미엄 포함. → 추천은 (a): 잠금은 '막(콘텐츠)'만.

## 2. 소유(해제) 상태 저장
- 1회 구매 = **기기/가정 단위**. 전역 키 `hp_unlocked`(프로필 접두 없음) → 한 번 사면 모든 아이 프로필에 적용(가족 공유 자연 충족).
- 이 값은 클라이언트라 그 자체로는 위조 가능 → **진짜 검증은 결제 계층이 담당**(§5).
- 핵심 훅: `unlockPremium()` / `isUnlocked()` / `accessAllowed()`.

## 3. 게이팅(잠금 발동) 지점
- 규칙: `현재 막 > FREE_MAX_ACT && !isUnlocked()` → 활동 대신 **페이월**.
- 발동 위치: ① 홈 렌더(renderMission) ② 막 경계 넘어갈 때(goNextLetter) ③ 활동 진입 시(resume/openTodayLetter).
- 무료 유지 영역(결정 2가 (a)일 때): 무료 막 글자의 복습·대시보드는 계속 접근.
- 엣지: 무료 막을 다 깬 아이가 3막 진입 시점에 페이월 1회 노출.

## 4. 페이월 UX (부모 대상, 아이 오결제 방지)
- 위치: 홈 미션 카드 자리 or 전용 화면. 부모 게이트(곱셈) 뒤에서만 결제 진입.
- 내용: "여기부터 전체 이용권 ✨" + 혜택(3~8막 전체·졸업까지·복습·대시보드·다자녀) + 가격 + [이용권 열기].
- 아이 화면엔 부담 없는 톤("하니가 다음 마을에서 기다려요 🔒"), 결제 문구는 부모 게이트 뒤.

## 5. 결제(해제) 계층 — 2단계 로드맵
- **A. 지금(구조/소프트런칭)**: `unlockPremium()` 훅 + 임시 수단(부모 확인 + 이용권 코드). 코드 방식은 복제에 약함 → 소프트런칭/지인 배포용.
- **B. 실제(권장)**: **스토어 인앱결제(비소모성)** 성공 콜백 → `unlockPremium()` 직접 호출. **서버 불필요**(스토어가 소유·복원 검증). = 가장 안정적.
  - 기기 변경/재설치: 스토어 '구매 복원(restore)'으로 재구매 없이 해제.
- **C. 대안(웹, 스토어 없이 빠르게)**: 토스페이먼츠 결제링크 → 유니크 코드 발급(경량: 구글시트/서버리스) → 앱에 코드 입력. 스토어 수수료·심사 없음, 대신 코드 관리·복제 리스크.
- ⚠️ **결정 3 — 결제 경로**: B(스토어 IAP, 안정·권장) / C(웹 코드, 빠름) / 둘 다(안드로이드는 B, 웹은 C).

## 6. 유통 로드맵(패키징)
- **Android**: 현재 PWA를 **TWA(Bubblewrap)**로 감싸 Play 등록 + Play Billing 비소모성 상품.
- **iOS**: **Capacitor** 래핑 + StoreKit 비소모성. Apple 개발자 계정($99/년).
- 준비물: 개발자 계정(Google $25 1회 / Apple $99/년), 앱 서명, 스토어 리스팅(스크린샷·설명), 심사(키즈 카테고리 규정), 상품(가격) 등록.
- 서버 운영비: **0**(앱은 정적, 소유 검증은 스토어). ← "안정적" 충족.

## 7. 가격·정책
- 저렴한 1회(예시): ₩3,900~5,900 (스토어 수수료 반영해 책정). ⚠️ **결정 4 — 가격대**.
- 필요 문서(유료배포_검토.md 연계): 개인정보처리방침·이용약관·환불정책, 국내 판매 시 사업자등록/통신판매업.
- 가족 공유: 전역 unlock으로 한 번 구매 = 온 가족.

## 8. 엣지/마이그레이션
- 기존 사용자: 무료 막까지 그대로, 프리미엄 막 진입 시 페이월.
- ⚠️ **결정 5 — 베타 사용자 예우**: 이미 3막+ 진행한 기존 사용자에게 무료 유지(grandfather)? 아니면 동일 적용?
- 기기 변경: 스토어 IAP면 복원 지원. 웹 코드면 코드 재입력.
- 무료↔프리미엄 전환 시 진도·복습 데이터는 그대로(잠금은 '접근'만 제어).

## ✅ 확정 결정 (2026-07-08)
1. **무료 경계: 2막까지** (1막 모음 + 2막 자음). 3막부터 프리미엄.
2. **다자녀만 유료**: 복습(SRS)·부모 대시보드는 무료 유지. **추가 프로필(2번째 아이부터)은 프리미엄**. (첫 프로필 무료)
3. **결제: 스토어 인앱결제(비소모성)**. 서버 없음.
4. **가격: ₩5,000** (1회).
5. **소유자(본인 기기)는 무료, 나머지 유료**: 내 기기는 unlocked 프리셋, 새 설치는 잠금.

## 🏗 확정 구조 (이 구조로 구현)

### 프리미엄 = 하나의 1회 구매로 아래 둘 다 해제
- **콘텐츠 게이트**: 현재 막 > 2 && !unlocked → 페이월. (3~8막 잠금)
- **프로필 게이트**: 미구매 상태에서 2번째 프로필 추가 시 → 페이월. (첫 아이 무료, 둘째부터 유료)

### 무료로 항상 열림
- 1~2막 전체 여정, 복습(SRS), 부모 대시보드, 첫 프로필 1개.

### 소유 상태
- 전역 플래그 `hp_unlocked`(프로필 접두 없음, 기기/가정 단위). 한 번 구매 = 모든 것 해제.
- 핵심 훅: `isUnlocked()` / `unlockPremium()`(스토어 결제 성공이 호출) / `accessAllowed()`(콘텐츠) / `canAddProfile()`(프로필).
- 소유자 예우: 내 기기는 `hp_unlocked=1` 프리셋(그랜드파더). 배포본 기본값은 잠금.

### 페이월 UX
- 부모 게이트(곱셈) 뒤. 혜택(3~8막 전체·졸업까지·다자녀) + ₩5,000 1회 + [이용권 열기 → 인앱결제].
- 아이 화면 톤: "다음 마을은 잠겨 있어요 🔒 (부모님과 함께 열어요)".

### 결제 계층(추상화)
- 지금 구조 단계: `unlockPremium()` 훅 + 임시 해제(부모확인/코드) — 내부 테스트용.
- 실제: Play Billing / StoreKit 비소모성 상품 구매 성공 → `unlockPremium()`. 재설치는 '구매 복원'.

## 구현 순서 (구조 확정 후)
1. **클라이언트 프리미엄 구조**(코드 없이 서버 불필요): `FREE_MAX_ACT=2`, `hp_unlocked` 전역, `isUnlocked/unlockPremium/accessAllowed/canAddProfile`, 페이월 UI, 콘텐츠·프로필 두 게이트, 임시 해제(부모확인). → 여기까지가 '구조 구현'.
2. **패키징**: TWA(Android)/Capacitor(iOS)로 래핑.
3. **스토어 인앱결제 연동**: 비소모성 상품 등록 + 결제 성공 → `unlockPremium()` + 복원.
4. **법적/스토어**: 개인정보·약관·환불, 사업자/통신판매업, 키즈 카테고리 심사.
